# Bright_TV_Viewership_Analysis
Bright TV case study analysing viewership trends and audience insights 

1. Overview of the Case Study
  This project focuses on analysing TV viewership data to understand audience behaviour. The goal was to identify patterns in how         different groups of people watch TV, including when they watch, what they watch, and how long they watch.

  Two datasets were provided:
  •	Viewership data 
  •	User profile data 
  These datasets were combined to create one final dataset for analysis.

3. Methodology (What Was Done)
   
  Data Preparation
  •	Joined the two datasets to create a complete dataset 
  •	Cleaned the data by removing errors and missing values 
  •	Standardised formats (e.g., converted time to South African time) 

  Feature Creation
  •	Created time buckets (morning, afternoon, evening, night) 
  •	Grouped users into age categories 
  •	Created weekday vs weekend categories

  Data Analysis
  •	Analysed viewing patterns across time, channels, and age groups 
  •	Used single-factor and two-factor analysis (e.g., Time vs Channel) 
  •	Identified trends in viewing duration and preferences 

  Data Visualisation
  •	Created charts and dashboards to present findings clearly 
  •	Used graphs to show patterns such as peak viewing times and popular channels

  Key Insights
  •	Viewership is higher during evening hours compared to other times 
  •	Certain channels perform better at specific times of the day 
  •	Different age groups have different viewing preferences 
  •	Weekends show different viewing behaviour compared to weekdays 
  •	Some users have consistently longer viewing durations than others 

3. Summary of Findings
The analysis showed clear patterns in TV viewership based on time, channel, and audience group. By combining and analysing the         datasets, it was possible to understand audience segmentation and viewing behaviour. These insights can help improve content     scheduling   and target specific audiences more effectively.

4. Tools Used
  •	Google BigQuery – for data storage, querying, and analysis 
  •	SQL – for data cleaning, joining, and transformation 
  •	Excel – for additional analysis and formatting 
  •	Google Looker Studio – for creating charts and presenting insights
